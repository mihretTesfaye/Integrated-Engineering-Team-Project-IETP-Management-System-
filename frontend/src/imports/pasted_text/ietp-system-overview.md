1. Actors and Their Roles
The IETP Management System involves three primary actors. Each actor has a distinct set of responsibilities within the system, reflecting their role in the real-world IETP lifecycle.

Actor	Role in System	Responsibilities
Administrator	System coordinator who manages the overall IETP workflow	•  Forms project groups (5–9 members, discipline mix)
•  Assigns 1advisor to each group
•  Creates and manages all user accounts and roles
•  Oversees project workflow from registration to archiving
•  Manages the project archive
Student	Primary project participant who carries out all project activities	•  Submits project ideas (Form A, SDG alignment, sketches)
•  Prepares and submits proposals and final reports
•  Logs milestones and weekly progress updates
•  Uploads all project documents and media files
•  Browses the project archive for reference
Advisor	Faculty member who supervises and evaluates assigned project groups	•  Reviews and approves project proposals
•  Monitors weekly progress and meeting records
•  Reviews submitted documents and provides feedback
•  Records formal evaluation scores (62.5% of final grade)

 
2. Core System Modules
The system is structured around six core modules, each responsible for a distinct functional area of the IETP lifecycle.

Module 1:  User Management
Handles all user-related operations including registration, authentication, and role-based access control.
•  User registration and secure login
•  Role assignment: Student, Advisor, Administrator
•  Profile management and password reset
•  Role-based access control (each role sees only relevant data)

Module 2:  Project Management
Manages the core project lifecycle from group creation through proposal submission and report upload.
•  Admin creates project groups (5–9 members, discipline mix)
•  Students submit project ideas (Form A, SDG alignment, concept sketches)
•  Proposal preparation and advisor approval workflow
•  Report upload at each project stage

Module 3:  Document Repository
Centralized storage for all project-related files, organized and searchable throughout the project lifecycle.
•  Centralized file storage for all project documents
•  Files organized by group, department, and academic year
•  Supports proposals, reports, presentations, drawings, videos, and posters
•  Search functionality by keyword, file type, group, or uploader

Module 4:  Evaluation & Scoring
Manages the formal assessment process including digital score entry, grade breakdowns, and performance summaries.
•  Digital score entry for advisors (62.5%) and peer evaluation (5%)
•  Prototype demonstration score recording (20%)
•  Oral presentation scoring (5%)
•  Grade summaries and performance reports per group



Module 5:  Progress Tracking
Enables continuous monitoring of project development through milestone logging, weekly updates, and meeting records.
•  Milestone logging with dates and descriptions
•  Weekly progress update submissions by students
•  Meeting records and e-portfolio documentation
•  Advisor view of all group progress in one dashboard

Module 6:  Project Archive
Read-only library of completed projects, indexed for discovery by future students and faculty.
•  Searchable archive by title, year, department, or keyword
•  Stores final reports, slides, and supporting materials
•  Prevents repetition of similar project ideas across academic years
•  Accessible to all users for reference and learning
